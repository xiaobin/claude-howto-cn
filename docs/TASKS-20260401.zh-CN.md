## 阶段 1 — 基础设施和薄弱模块（2026年4月）

**目标里程碑：M1**

### 支柱 4：新人入门

- [ ] **T-001** — 编写 `scripts/quickstart.sh`
  - 检测 `~/.claude/` 目录（若缺失则创建并经用户确认）
  - 将第一个斜杠命令 + CLAUDE.md + 技能复制到用户设置
  - 追加代理发现行到 CLAUDE.md，链接到 AGENT-INDEX.md
  - 幂等：跳过已有文件并发出警告，永不覆盖
  - 完成后打印后续步骤

- [ ] **T-002** — 创建 `QUICKSTART.md`（前15分钟视觉指南）
  - 将带注释的终端步骤作为代码块（尽可能避免 PNG 截图；首选 ASCII 或 Mermaid）
  - 记录哪些命令产生哪些视觉效果，以便重新捕获

- [ ] **T-003** — 为所有 10 个模块 README 添加难度徽章
  - 格式：在每个模块 README 顶部添加 `> 🟢 **Beginner** | ⏱ 30 minutes`
  - 在徽章下方添加"What you'll build"预览

### 支柱 7：活课程

- [ ] **T-004** — 创建 `WHATS-NEW.md`
  - 格式：`## CC vX.X — 日期`，每个功能变更一个条目
  - 为当前 Claude Code 版本至少添加一个条目

- [ ] **T-005** — 为所有 10 个模块 README 添加版本徽章
  - 格式：`> ✅ Verified against Claude Code **vX.X.X** · Last verified: YYYY-MM-DD`
  - 在每个模块的 frontmatter 中添加 `cc_version_verified`

- [ ] **T-006** — 创建 `.github/workflows/staleness-check.yml`
  - 执行计划：每周一 UTC 09:00
  - 从 frontmatter 读取每个模块的 `last_verified` 日期（使用 `yq` 或 10 行 Python 脚本）
  - 为 30+ 天未验证的模块创建 GitHub Issue
  - 若已存在同标题的开放 Issue 则跳过

### 支柱 6：内容质量（CLAUDE.md）

- [ ] **T-007** — 为项目创建根目录 `CLAUDE.md`
  - 将项目的自身配置作为最佳实践示例
  - 记录贡献约定、模块结构和维护工作流

### 支柱 2：AI 代理索引（生成器）

- [ ] **T-008** — 编写 `scripts/build-agent-index.py`
  - 读取所有 10 个模块 README 文件
  - 提取：标题（→ 主题）、代码块（→ 带语言标签的示例）、表格（→ 参考数据）
  - 输出 `agent-manifest.json`（使用 CEO 计划中定义的模式）
  - 输出 `AGENT-INDEX.md`（人类可读的摘要）
  - 验证引用指向实际文件

### 模块 08-checkpoints：完整深度整理（311 → 800+ 行）

- [ ] **T-009** — 使用真实场景重写模块介绍
  - 示例："你的重构刚刚破坏了 3 个地方。检查点如何拯救你..."

- [ ] **T-010** — 添加检查点策略决策树（Mermaid 流程图）
  - "我想做 X" → 沿箭头走 → 落在检查点模式上

- [ ] **T-011** — 添加 3 个工作流模板
  - 安全重构工作流
  - 使用检查点的 A/B 测试
  - 灾难恢复工作流

- [ ] **T-012** — 添加与规划模式的集成部分

- [ ] **T-013** — 添加检查点生命周期的 Mermaid 图表

- [ ] **T-014** — 添加 2+ 个"Try It Now"块，包含可验证命令
  - 每个块：上下文句子、要运行的命令、预期输出描述

- [ ] **T-015** — 添加"模式和配方"部分，包含 2+ 个命名模式
  - 每个模式：问题、解决方案代码、何时使用、何时不用

### 模块 06-hooks：完整深度整理

- [ ] **T-016** — 使用真实场景重写模块介绍
  - 示例："CI 在 20 分钟后捕获了 lint 错误。钩子在 2 秒内就能捕获..."

- [ ] **T-017** — 添加钩子决策树（Mermaid 流程图）
  - "我想在 Y 发生时触发 X" → 沿箭头走 → 落在钩子类型上

- [ ] **T-018** — 添加 5 个完整钩子示例，包含完整脚本（每个约 80 行）
  1. 写入时自动格式化（prettier/black）
  2. 提交时安全扫描（trufflehog/bandit）
  3. 文件更改时运行测试
  4. 会话结束时通知
  5. 提示词验证（阻止危险模式）

- [ ] **T-019** — 添加 2+ 个"Try It Now"块，包含可验证命令

- [ ] **T-020** — 添加"模式和配方"部分，包含 2+ 个命名模式

**M1 检查点：基础设施上线，2 个最弱模块完全升级**
