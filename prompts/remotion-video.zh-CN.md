# 自动化工作流

**核心原则：**

- **先探索：** 始终首先自动探索代码库，收集产品信息。不要以关于产品的问题开始。
- **先问后规划：** 探索后，在创建规划前向用户展示发现并询问视频偏好（尺寸、风格、时长、自定义项），**然后再开始规划**。
- **产品 URL 优先：** 当找到或提供了产品 URL 时，它作为**主要事实来源**。产品页面的信息优先于代码库发现。
- **价值优于技术：** 聚焦于价值主张、客户收益和功能（用户获得什么），而非技术规格或实现细节。
- **以客户为中心：** 强调产品如何解决问题、改善生活或为用户带来收益。
- **自主执行：** 用户确认偏好后，自主完成规划和实现，无需进一步批准请求。

## 第一阶段：自主资源发现

**目标：** 自动探索代码库，在不询问用户的情况下收集所有可用的产品信息。

**行动：**

1. **首先自动探索代码库：**
   - 搜索 `README.md` 获取产品描述和价值主张
   - 检查 `package.json` 获取产品名称、描述、主页 URL
   - 在 `/assets`、`/public`、`/static`、`/images` 目录中查找品牌资产
   - 从 CSS/Tailwind 配置文件中提取配色方案
   - 查找任何现有的营销文案或文档
   - 在配置文件、环境变量或文档中查找产品 URL

2. **如果找到产品 URL，立即获取：**
   - 使用 WebFetch 从产品页面提取信息
   - 产品页面信息优先于代码库发现
   - 提取所有价值主张、特性和品牌信息

3. **综合所有收集到的信息：**
   - 产品名称和描述
   - 价值主张
   - 关键特性和收益
   - 品牌颜色和风格
   - 目标受众（从语气推断）
   - 任何现有资产或媒体

4. **为缺失信息应用智能默认值：**
   - **视频格式：** 横屏 1920x1080（YouTube/网页优化）
   - **时长：** 30 秒（适合大多数平台）
   - **风格：** 现代、简洁、专业（基于品牌）
   - **品牌颜色：** 使用提取的颜色或互补的现代调色板

5. **仅在以下情况下询问用户（探索后）：**
   - 无法确定产品名称或找不到任何产品信息
   - 无法找到或访问产品 URL
   - 存在关键歧义（例如 B2B vs B2C 显著影响信息传递）
   - 存在需要澄清的冲突信息

**重要提示：** 完全静默自主地完成整个探索过程。不要问"我需要什么来开始"或列出需求。只有在真正必要时才打断用户。

**输出：** 立即进入第二阶段，携带所有收集到的信息。

---

## 第二阶段：信息分析与深度挖掘

**目标：** 分析收集到的信息，提取用于视频创建的关键洞察。

**行动：**

1. **审查第一阶段收集的所有信息：**
   - 产品页面内容（如果找到了 URL 并已获取）
   - 代码库发现（README、package.json、资产等）
   - 任何品牌指南或营销材料

2. **提取并优先排序（聚焦于价值，而非技术）：**
   - **价值主张**（主要焦点）— 对客户的主要收益
   - **客户收益**（用户获得什么）— 如何改善他们的生活
   - **关键特性**（以收益描述，而非技术规格）
   - **独特卖点** — 与其他产品有何不同/更好
   - **使用场景** — 真实世界应用
   - **品牌标识**（颜色、字体、风格、语气）
   - **目标受众洞察**（这是为谁服务的）
   - **情感诉求**和信息传递（人们为什么在乎）

3. **静默智能填补空白：**
   - 如果价值主张不明确，从特性禾目标受众推断
   - 如果目标受众不明确，从产品类型和信息传递语气推断
   - 如果品牌颜色缺失，创建互补的现代调色板
   - 避免技术实现细节，除非是面向用户的

4. **仅在以下情况下要求澄清：**
   - 存在多个相互冲突的价值主张
   - 无法确定产品是 B2B 还是 B2C（显著影响信息传递）
   - 目标受众真正不明确

**输出：** 对产品价值、收益和品牌有清晰理解，准备创建视频。

---

## 第三阶段：展示发现并收集用户偏好

**目标：** 分享发现的内容，并在规划前获取用户对视频偏好的输入。

**行动：**

1. **展示已发现信息的摘要：**

   ```text
   📊 已发现的信息

   产品：[名称]
   价值主张：[对客户的主要收益]
   关键特性：[2-3 个主要收益]
   品牌颜色：[提取或建议的颜色]
   目标受众：[这是为谁服务的]
   ```

2. **询问用户偏好（继续前必须）：**

   使用清晰简洁的格式：

   ```text
   在我创建视频之前，请告诉我你的偏好：

   1. **视频尺寸/格式：**
      - 横屏 (1920x1080) - YouTube、网站
      - 竖屏 (1080x1920) - TikTok、Instagram Reels
      - 方形 (1080x1080) - Instagram 动态

   2. **视频时长：**
      - 15 秒 - 快速社交媒体广告
      - 30 秒 - 标准推广视频
      - 60 秒 - 详细功能展示
      - 自定义时长

   3. **视频风格：**
      - 现代简约 - 简洁、Apple 风格美学
      - 活力大胆 - 快节奏、社交媒体风格
      - 专业商务 - 商业导向
      - 自定义风格（描述你的愿景）

   4. **还有什么需要突出或自定义的吗？**
      （特定功能、信息传递、颜色等）
   ```

3. **等待用户回复** 后再进入第四阶段。

4. **确认偏好并响应：**
   - 总结用户的选择
   - 应用任何自定义要求
   - 以确认的方向开始结构设计

**输出：** 用户确认的视频规格，准备进入规划阶段。

---

## 第四阶段：结构设计（确认后）

**目标：** 基于用户确认的偏好，使用三幕格式创建引人入胜的视频结构。

**行动：**

1. **根据用户确认的偏好设计视频结构：**

   ```text
   🎬 视频结构

   第一幕：钩子（0-5 秒）
   - [吸引眼球的视觉概念]
   - [大胆的动画入场]
   - [引人注目的标题/问题]

   第二幕：价值展示（中间部分）
   - [展示关键收益的实际应用]
   - [客户价值的视觉叙事]
   - [2-3 个以收益呈现的特性亮点]

   第三幕：行动号召（最后部分）
   - [清晰的 CTA 配合品牌强化]
   - [令人难忘的结尾视觉]
   - [流畅的退出动画]
   ```

2. **应用用户偏好：**
   - 使用指定的视频尺寸/格式
   - 匹配所选风格（简约/活力/专业）
   - 根据指定时长调整时间
   - 合并任何自定义要求

3. **基于以下内容做出创意决策：**
   - 产品价值主张（什么让它引人注目）
   - 目标受众（什么能引起他们共鸣）
   - 用户的风格偏好
   - 品牌个性（视觉和语气一致性）

4. **简要展示结构** 然后自动进入第五阶段。

**输出：** 完整的视频结构，准备进入实现规划。

## 第五阶段：技术架构

**目标：** 设计实现架构并直接开始构建。

**行动：**

1. **静默设计** 组件架构：
   - 工具函数（缓动、动画辅助、颜色工具）
   - 可复用组件（AnimatedTitle、FeatureHighlight 等）
   - 场景组件（Hook、Demo、CTA 场景）
   - 主合成结构（Video.tsx、Root.tsx）

2. **规划技术细节：**
   - 动画时序和缓动曲线
   - 调色板实现
   - 字体层级
   - 图标和资产策略
   - 序列时序分解

3. **不请求批准，直接进入第六阶段实现。**

**输出：** 内部技术蓝图，准备立即实现。

---

## 第六阶段：实现

**目标：** 自主构建完整的 Remotion 视频项目。

**约束与技术栈：**

1. **框架：** Remotion (React)
2. **样式：** Tailwind CSS（通过 `className` 或标准样式对象）
3. **动画：** 使用 `spring`、`interpolate` 和 `useCurrentFrame` 实现流畅运动
4. **代码风格：** 模块化组件。不要把所有代码都堆在 `Root.tsx` 里
5. **最佳实践：**
   - 没有什么应该是静态的。所有内容都必须有入场（透明度/缩放/滑动）和退场
   - 需要时使用 Lucide-React 图标
   - 使用标准字体但大量样式化（粗体、字间距收紧）
   - 不要使用外部图片，除非是占位符（如 `https://placehold.co/600x400`）或用户提供资产

**行动：**

1. **按此顺序构建完整项目结构：**
   - 工具函数（缓动、动画辅助、颜色工具）
   - 可复用组件（AnimatedTitle、FeatureHighlight、过渡效果）
   - 场景组件（HookScene、DemoScene、CTAScene）
   - 主合成（Video.tsx 配合序列）
   - 根配置（Root.tsx 配合正确的注册）

2. **静默高效地工作：**
   - 创建所有文件，不叙述每个步骤
   - 根据收集到的信息做出设计决策
   - 使用专业动画原则
   - 确保场景间流畅过渡

3. **实现完成后自动进入第七阶段。**

**输出：** 完整、可用于生产的 Remotion 项目代码。

   ```bash

   npm run dev

   # 渲染最终视频
   npm run build
   npx remotion render Video out/video.mp4

   # 特定编码/设置
   npx remotion render Video out/video.mp4 --codec h264
   ```

2. **交付摘要：**
   - 简要描述创建的内容
   - 视频的关键特性
   - 视频规格（时长、格式、尺寸）
   - 任何值得注意的设计决策

3. **如需更改，用户可以请求：**
   - 时序调整
   - 动画修改
   - 内容更新
   - 风格微调

**输出：** 完整的 Remotion 项目，包含清晰的渲染说明，可投入使用。

# 质量标准

在所有阶段中，保持这些标准：

**视觉质量：**
- 专业级动画（流畅、有目的、与品牌一致）
- 一致的间距和对齐
- 可读性强的字体和适当的对比度
- 内聚的配色使用

**技术质量：**
- 清晰、模块化的代码架构
- 性能优化（流畅 30fps 播放）
- 正确使用 Remotion API（spring、interpolate、Sequence）
- 类型安全（使用 TypeScript 时）

**创意质量：**
- 清晰的叙事结构
- 吸引眼球的开场
- 强有力的行动号召
- 令人难忘的视觉瞬间
